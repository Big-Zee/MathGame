# Data Model: Practice Mode

**Branch**: `003-practice-mode` | **Date**: 2026-04-29
**Plan**: [plan.md](./plan.md) | **Research**: [research.md](./research.md)

All Practice Mode state is held in plain JavaScript objects and discarded on exit.
Nothing is written to `localStorage`. The main game's entities (`GameConfig`, `Question`,
`GameSession`, `PlayerResult`) are **unchanged**.

---

## New Export: PracticeRanges

Immutable constant. Defines the operand constraints for each difficulty × operation combination.
Lives in `js/math-engine.js` alongside `GameConfig`.

```js
export const PracticeRanges = {
  easy: {
    add: { aMin: 1,  aMax: 10, bMin: 1, bMax: 10  },   // sums up to 20
    sub: { aMin: 2,  aMax: 20, bMin: 1, bMax: 9   },   // results 1–19
    mul: { aMin: 2,  aMax: 10, bMin: 2, bMax: 5   },   // 2–5 times tables
    div: { aMin: 4,  aMax: 50, bMin: 2, bMax: 5   },   // divisors 2–5
  },
  medium: {
    add: { aMin: 1,  aMax: 50, bMin: 1, bMax: 49  },   // sums up to 99
    sub: { aMin: 5,  aMax: 50, bMin: 1, bMax: 44  },   // results 1–49
    mul: { aMin: 2,  aMax: 10, bMin: 2, bMax: 10  },   // 2–10 times tables
    div: { aMin: 4,  aMax: 100, bMin: 2, bMax: 10 },   // divisors 2–10
  },
  hard: {
    add: { aMin: 1,  aMax: 99, bMin: 1, bMax: null },  // = main game ranges
    sub: { aMin: 10, aMax: 99, bMin: 1, bMax: null },
    mul: { aMin: 2,  aMax: 12, bMin: 2, bMax: 12  },
    div: { aMin: 4,  aMax: 100, bMin: 2, bMax: 12 },
  },
};
```

**Note on `bMax: null`**: Hard tier inherits the main game's dynamic bound
(`100 − a` for add, `a − 1` for sub) via the amended `generateQuestion` null-coalescing
guard (see research.md Decision 2).

## New Export: getPracticeConfig(operation, difficulty)

```js
export function getPracticeConfig(operation, difficulty) {
  return {
    ...GameConfig,
    numberRanges: { [operation]: PracticeRanges[difficulty][operation] },
  };
}
```

**Returns**: A config object compatible with `generateQuestion(operation, config, id)`.
**Immutable after construction** — safe to cache for the duration of a practice session.

## Amendment: generateQuestion (backward-compatible)

Two lines changed in the `add` and `sub` cases to respect `r.bMax` when non-null:

```js
// add case:
b = randInt(r.bMin, Math.min(r.bMax ?? Infinity, 100 - a));

// sub case:
b = randInt(r.bMin, Math.min(r.bMax ?? Infinity, a - 1));
```

Main game config has `bMax: null` → behavior is unchanged for the main game.

## PracticeSession

Mutable. Created by `newPracticeSession(operation, difficulty)` when the child confirms
difficulty selection. Replaced entirely when "Practise Again" is pressed.

```js
{
  operation:      string,    // 'add' | 'sub' | 'mul' | 'div'
  difficulty:     string,    // 'easy' | 'medium' | 'hard'
  config:         object,    // result of getPracticeConfig(operation, difficulty) — cached
  seenKeys:       Set,       // Set<string> of "a-operation-b" keys seen this session
  totalAnswered:  number,    // 0+; increments on every Submit
  totalCorrect:   number,    // 0+; increments on correct Submit
  currentStreak:  number,    // consecutive correct answers; resets to 0 on wrong
  bestStreak:     number,    // peak currentStreak achieved; never decreases
  currentQuestion: Question | null,  // the question currently on screen
}
```

**State transitions**:

```
null
  │ [child confirms difficulty]
  ▼
practiceSession (initialized)
  │ [Submit — correct]  → totalAnswered++, totalCorrect++, update streak, load next Q
  │ [Submit — wrong]    → totalAnswered++, streak → 0, reveal answer, load next Q
  │ [Stop Practising]   → derive PracticeSummary, show summary screen
  │ [Practise Again]    → null → re-initialize with same operation+difficulty
  │ [Back to Start]     → null
  ▼
null
```

## PracticeQuestion

Identical to the existing `Question` shape; generated by the existing `generateQuestion()`.
The `choices` field is generated but **not used** — Practice Mode displays a text input,
not multiple-choice buttons. The `id` field is always `0` (position within session is not tracked).

```js
{
  id:        0,         // always 0 for practice questions
  operation: string,    // 'add' | 'sub' | 'mul' | 'div'
  symbol:    string,    // '+' | '−' | '×' | '÷'
  a:         number,
  b:         number,
  answer:    number,    // correct integer result; ≤ 100
  choices:   number[],  // generated but not rendered in Practice Mode
}
```

**Deduplication key**: `` `${q.a}-${q.operation}-${q.b}` `` — added to `seenKeys` on use.

## PracticeSummary

Derived from `PracticeSession` when "Stop Practising" is tapped. Not stored; computed inline.

```js
{
  totalAnswered:  number,   // from practiceSession.totalAnswered
  totalCorrect:   number,   // from practiceSession.totalCorrect
  accuracyPct:    number,   // Math.round(totalCorrect / totalAnswered * 100) || 0
  bestStreak:     number,   // from practiceSession.bestStreak
  messageTier:    string,   // 'master' | 'amazing' | 'good' | 'keep-going'
}
```

**Message tier mapping**:

| accuracyPct | messageTier | Display text |
|-------------|-------------|--------------|
| 90–100 | `'master'` | "You're a Math Blaster master! 🏆" |
| 70–89 | `'amazing'` | "Amazing work, keep it up! 🌟" |
| 50–69 | `'good'` | "Good effort! A little more practice and you'll ace it! 💪" |
| 0–49 | `'keep-going'` | "Keep going, every question makes you smarter! 🧠" |

**Edge case**: `totalAnswered === 0` → `accuracyPct = 0` → tier `'keep-going'`. Both action
buttons remain functional.

## Entity Relationships

```
PracticeRanges (singleton, read-only)
  └─ read by ─→ getPracticeConfig(op, diff)
                    │
                    └─ cached in ─→ PracticeSession.config
                                        │
                    generateQuestion(op, config, 0)
                                        │
                                        └─→ PracticeQuestion
                                                 │
                    dedup check ─→ PracticeSession.seenKeys
                                        │
                    answer evaluated ─→ PracticeSession.{totalAnswered, totalCorrect,
                                                          currentStreak, bestStreak}
                                        │
                    [Stop] derive ─→ PracticeSummary (ephemeral, not stored)
```

No writes to `localStorage`. No reads from `GameSession` or `PlayerResult`.
