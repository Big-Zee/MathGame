# Data Model: Math Quiz Game

**Branch**: `001-math-quiz-game` | **Date**: 2026-04-28
**Plan**: [plan.md](./plan.md) | **Research**: [research.md](./research.md)

All state is held in plain JavaScript objects. No external storage schema is required.
`localStorage` stores one scalar value (high score integer).

---

## GameConfig

Runtime configuration object. Defined once at the top of `js/math-engine.js` and exported.
Never mutated during a session.

```js
{
  totalQuestions:  10,          // questions per round
  timerSeconds:    10,          // seconds allowed per question
  basePts:         10,          // points for any correct answer
  streakPts:       15,          // points per correct answer when streak active
  streakThreshold: 3,           // consecutive correct answers to activate streak

  starThresholds: {
    two:   70,   // score >= 70 → 2 stars
    three: 130,  // score >= 130 → 3 stars (requires ~9 correct with streak)
  },

  operations: ['add', 'sub', 'mul', 'div'],  // all four included in every round

  numberRanges: {
    add: { aMin: 1,  aMax: 99, bMin: 1,  bMax: 99  },
    sub: { aMin: 10, aMax: 99, bMin: 1,  bMax: null },  // bMax = a-1 (ensured at gen time)
    mul: { aMin: 2,  aMax: 12, bMin: 2,  bMax: 12  },
    div: { aMin: 4,  aMax: 144, bMin: 2, bMax: 12  },   // a = b * quotient (whole number)
  },
}
```

**Validation rules**:
- `sub`: `b < a` always enforced so the result is a positive integer
- `div`: `a` is computed as `quotient * b` where `quotient ∈ [1, 12]` and `b ∈ [2, 12]`

## Question

Represents one quiz question. Generated by `generateQuestion()` in `math-engine.js`.
Immutable after creation.

```js
{
  id:        number,    // 0-based index within the round (0–9)
  operation: string,    // 'add' | 'sub' | 'mul' | 'div'
  symbol:    string,    // '+' | '−' | '×' | '÷'  (display character)
  a:         number,    // left operand
  b:         number,    // right operand
  answer:    number,    // correct result (always a positive integer)
  choices:   number[],  // exactly 4 unique integers, shuffled, includes answer
}
```

**Invariants**:
- `choices.length === 4`
- `choices` contains no duplicates
- `choices` includes `answer`
- All `choices` are positive integers

---

## GameSession

Mutable state object holding the current round's progress. Created fresh when "Play" is pressed.
Never persisted (lives only for the duration of the round).

```js
{
  score:         number,    // 0..200 (10 questions × max 15pts + rounding overhead)
  lives:         number,    // 3 → 0; game ends when reaches 0
  streak:        number,    // consecutive correct answers; resets to 0 on wrong
  questionIndex: number,    // 0-based; current question (0–9)
  questions:     Question[],// full array of 10 pre-generated questions
  timerTicks:    number,    // 0–100 (tenths of seconds); counts down from 100
  timerHandle:   number|null, // setInterval handle; cleared on question end
  phase:         string,    // 'question' | 'feedback' | 'done'
}
```

**State transitions** (see `ui-state-machine.md`):
- `phase: 'question'` → player interaction enabled, timer running
- `phase: 'feedback'` → choices disabled, emoji shown, auto-advance after 1 000ms
- `phase: 'done'` → timer cleared, Results screen rendered

---

## PlayerResult

Derived from `GameSession` at round end. Passed to the Results screen renderer.
Not persisted directly (only `score` is compared to `localStorage` high score).

```js
{
  score:        number,   // final score
  correctCount: number,   // 0–10
  stars:        number,   // 1 | 2 | 3
  highScore:    number,   // stored high score (updated if score > previous)
  newHighScore: boolean,  // true if this round beat the previous high score
}
```

---

## localStorage Schema

| Key | Type | Description |
|-----|------|-------------|
| `mathgame_highscore` | integer string | All-time high score. Defaults to `0` if absent or invalid. |

No other keys are written. The game reads and writes only this one value.

---

## Entity Relationships

```
GameConfig (singleton, read-only)
  └─ used by ─→ generateRound() → Question[10]
                                       │
                              held in ─→ GameSession
                                       │
                              produces ─→ PlayerResult
                                                │
                                     writes ─→ localStorage (highScore only)
```
